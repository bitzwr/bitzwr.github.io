Java日志工具-log4j

Dec 13, 2015 | Java

日志是记录软件运行状态的工具，是软件中不可或缺的部分。Apache的开源项目log4j是一个功能强大的日志组件，比jdk提供的日志组件更加方便。

概要

Log4j主要分为3部分，Logers、Appenders、Layouts。其中Logger用来设置日志的类型和级别，Appender决定日志的输出地方，Layout用于设置日志信息的格式。

Logger

日志的级别包括: TRACE、DEBUG、INFO、WAEN、ERROR、FATAL。日志级别递增，在开发调试过程中一般使用DEBUG级别，生产环境中使用INFO级别。

Appender

  • ConsoleAppender (控制台)
  • FileAppender (文件)
  • DaliyRollingFileAppender (每天产生一个日志文件)
  • RollingFileAppender (文件大小到达指定尺寸时产生一个新文件)
  • WriteAppender (将日志信息以流格式发送到任意指定的地方)
  • JDBCAppender (将日志信息保存到数据库中)

常用的是前四个, 具体使用方法参照示例部分。

Layout

  • HTMLLayout (以HTML表格形式布局)
  • SimpleLayout (简单布局:包含日志信息的级别和信息字符串)
  • TTCCLayout (包含日志产生时间、执行者、类别等信息)
  • PatternLayout (可以灵活地指定布局格式，最为常用)

布局信息

通常适用org.apache.log4j.PatternLayout来指定。

  • %c 输出日志信息所属类的全名
  • %d 输出日志的时间,默认格式为ISO8601,可以在其后指定格式,%d{yyyy-MM-dd HH:mm:ss:SSS},输出结构: 2016-01-24 15:53:14:999
  • %F 输出日志所属类的类名
  • %l 输出日志发生的位置，效果等同 %c.%M(%F:%L)
  • %L 输出日志的语句在代码中的行号
  • %m 输出代码中指定的信息,message
  • %M 输出日志所在的方法名, 格式: <methodName>
  • %n 输出一个行号符,Win平台下为”\r\n”, Unix平台为”\n”
  • %p 输出日志级别
  • %r 输出自应用启动到输出该日志所消耗的毫秒数
  • %t 输出日志的线程名
